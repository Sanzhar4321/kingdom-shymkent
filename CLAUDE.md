# Kingdom Shymkent — лендинг

Один автономный файл: [dist/index.html](dist/index.html)
Стек: чистый HTML + CSS + vanilla JS, GSAP с CDN (анимации появления секций, счётчики статистики, плавающие "орбы" фона).

## Запуск

```bash
cd dist && python3 -m http.server 8000
```
Открыть http://localhost:8000

## Проверка перед сборкой

```bash
node verify.js
```
Скрипт [verify.js](verify.js) проверяет:
- парность HTML-тегов
- наличие файлов лого `dist/logo-crown.png` и `dist/logo-full.png`
- подключение лого-меток (`logo-mark--crown` в nav/футере, `logo-mark--full` в hero)
- равное количество `data-kaz` и `data-ru` атрибутов

## Двуязычность (ҚАЗ / РУС)

- По умолчанию язык — казахский (`localStorage` пуст → `kaz`).
- Переключатель в навигации: кнопки `.lang-btn` с `data-lang="kaz"` / `data-lang="ru"`.
- Любой текстовый элемент, который должен переводиться, получает класс `t` и атрибуты `data-kaz="..."` и `data-ru="..."`.
- Функция `setLanguage(lang)` (в `<script>` внизу файла):
  - проходит по всем `.t`,
  - для `<input>`/`<textarea>` подставляет значение в `placeholder`,
  - для остальных элементов — в `textContent`,
  - подсвечивает активную кнопку переключателя,
  - сохраняет выбор в `localStorage.kingdom_lang`.
- При загрузке страницы читается `localStorage.kingdom_lang`, иначе — `kaz`.

**Важно:** при добавлении нового текста всегда добавляй оба атрибута (`data-kaz` и `data-ru`) на один и тот же элемент — иначе `verify.js` сообщит о несовпадении количества.

## Структура секций (по образцу лидеров рынка: InterPress, JSI)

`#top` (hero + бегущая строка) → блок цифр → `#courses` (5 курсов, клик по курсу = модалка с описанием курса + кнопка записи) → `#leveltest` (тест уровня, лид-магнит) → `#about` (тёмный блок, преимущества) → `#teachers` (горизонтальный свайп, как отзывы) → `#reviews` (5, горизонтальный свайп) → `#faq` (аккордеон, 5 вопросов) → `#form` (заявка → WhatsApp) → `#contacts` → footer.

## Тест уровня (`#leveltest`)

Интерактивный лид-магнит: определяет примерный уровень (A1–C1) английского **или** русского, в конце показывает результат + рекламу школы + кнопку записи на бесплатную консультацию (→ WhatsApp с уже подставленным уровнем).

- **Три состояния** в одной карточке `.quiz-card`, переключаются через `display` функцией `show('intro'|'play'|'result')`:
  - `#quizIntro` — выбор языка (две кнопки `.quiz-pick` с `data-test="en"`/`data-test="ru"`);
  - `#quizPlay` — вопрос (`#quizQuestion`), варианты (`#quizOptions`), прогресс (`#quizBar`), счётчик (`#quizCount`);
  - `#quizResult` — бейдж уровня (`#quizBadge`), название (`#quizLevelName`), счёт (`#quizScore`), описание (`#quizLevelDesc`), реклама (`#quizAd`), CTA `#quizConsult`, переотступ `#quizRetake`.
- **Вопросы** живут в JS-объекте `bank.en` / `bank.ru` (по 12 шт.) в блоке `/* ===== LEVEL TEST ===== */` внизu `<script>`. Формат: `{ q:"... ___ ...", opts:[4 варианта], c:индекс_правильного }`. Плейсхолдер `___` в тексте автоматически заменяется на `· · ·`. **Эти строки НЕ переводятся** через `data-kaz`/`data-ru` — содержание теста (англ./рус. предложения) одинаково в обоих языках интерфейса, поэтому на парность `verify.js` не влияет.
- **Локализуемый текст** (название уровня, описание, реклама, метка языка, подпись счёта) — в объекте `i18n.kaz` / `i18n.ru` внутри того же блока. При смене языка `setLanguage` обновляет статические `.t`, а отдельный слушатель на `.lang-btn` перерисовывает динамику теста (`renderQuestion` / `renderResult`).
- **Подсчёт уровня** — функция `levelFor(score)`: 0–3 → A1, 4–5 → A2, 6–8 → B1, 9–10 → B2, 11–12 → C1.
- **Как менять вопросы/уровни:** правьте `bank` (вопросы) и `i18n` (тексты результата). Количество вопросов задаётся `var TOTAL` (по умолчанию 12) — при изменении пересмотрите пороги в `levelFor`.

## Как работают заявки

Бэкенда нет — всё через WhatsApp (реально работает сразу):
- Все элементы с классом `.wa-link` (hero-кнопка, контакты, плавающая кнопка) открывают `wa.me/<WA_NUMBER>` с приветственным сообщением на текущем языке.
- Клик по строке курса открывает **модальное окно** `#courseModal` с описанием курса (заголовок, мета, абзац, 3 пункта). Тексты модалки живут в JS-объекте `courseInfo` в `<script>` (ключ = `data-course` строки курса, поля `title/meta/desc/points` с `kaz`/`ru`) — **не** через `data-kaz`/`data-ru`, на парность `verify.js` не влияют. Кнопка «Тегін сынақ сабағына жазылу / Записаться на бесплатный пробный урок» внутри модалки открывает WhatsApp с сообщением про этот курс. Закрытие: крестик, клик по фону, Esc. При смене языка открытая модалка перерисовывается.
- Форма `#leadForm` валидирует телефон (маска +7, 11 цифр) и открывает WhatsApp с готовой заявкой: имя, телефон, курс, время. После отправки показывается «Спасибо».

## Логотип

Реальный логотип-картинка (плетёная корона + KINGDOM), два PNG в `dist/`:
- `logo-crown.png` — только корона (для nav и футера), aspect ≈ 700/464;
- `logo-full.png` — корона + надпись KINGDOM (для hero), aspect ≈ 640/768.

Выводятся через CSS-маску, чтобы форма бралась из PNG, а цвет — золотой (заливка `linear-gradient` золотом):
```css
.logo-mark{ background:linear-gradient(135deg,var(--gold-soft),var(--gold) 65%);
  -webkit-mask:url(...) center/contain no-repeat; mask:url(...) center/contain no-repeat; }
```
- nav/футер: `<span class="logo-mark logo-mark--crown">` + текстовый `.brand-word` (KINGDOM SHYMKENT);
- hero: `<span class="logo-mark logo-mark--full">` (внутри `#heroCrest`, с золотым drop-shadow).

**Чтобы поменять лого:** заменить PNG в `dist/` (белая форма на прозрачном фоне — альфа-канал = форма; цвет даёт маска). Источник нарезался из `logo-full.png` скриптом на PIL (crop по альфе + разрез на корону/надпись). Favicon — отдельный инлайн-SVG в `<link rel="icon">` (повторяет 8 линий короны золотом на тёмном квадрате).
